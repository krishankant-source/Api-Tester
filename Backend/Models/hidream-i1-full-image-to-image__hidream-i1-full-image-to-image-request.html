Request Parameters - HiDream I1 Full (Image to Image) generate request

Field Type Required Default Description
prompt string Yes — Text description guiding the image transformation. Must describe the desired output style, content, or edit.
image_url string Yes — Publicly accessible URL of the source image to use as input for image-to-image generation.
negative_prompt string No "" Details to avoid in the generated image. Use to exclude unwanted elements, styles, or artifacts.
image_size enum or object No — Output image dimensions. Use enum values: square_hd, square, portrait_4_3, portrait_16_9, landscape_4_3, landscape_16_9. Alternatively, pass an object {width: number, height: number}. If omitted, uses input image dimensions.
num_inference_steps integer No 50 Number of diffusion denoising steps. Higher values improve quality but increase processing time. Range: 20–100.
seed integer No — Random seed for reproducible results. If omitted, a random seed is used.
guidance_scale float No 5 Classifier-free guidance scale. Controls how closely the output follows the prompt. Higher values increase prompt adherence. Range: 1–20.
sync_mode boolean No false If true, returns base64-encoded data URI instead of CDN URL. Use for low-latency, small-scale applications.
num_images integer No 1 Number of images to generate in a single request. Range: 1–4.
enable_safety_checker boolean No true Toggle the NSFW content safety filter. Disable only if you are certain your content is safe.
output_format enum No jpeg Output image format. Allowed values: jpeg, png.
loras array No [] List of LoRA adapters to apply during generation. Each item is an object with: path (string, required), weight_name (string, optional), scale (float, optional, default=1).
strength float No 0.75 Denoising strength for image-to-image generation. Controls how much the output deviates from the input. Higher values allow more transformation. Range: 0.1–1.0.

Request Status Workflow

Status values for tracking asynchronous processing.

Status Description
QUEUED Request accepted, waiting to be processed
PROCESSING Being processed by the model
COMPLETED Done — output contains the result
FAILED Failed — check error field for details
ERROR System error — request was not processed and will not be charged

Status Flow

QUEUED → PROCESSING → COMPLETED
                     ↘ FAILED
                     ↘ ERROR

Typical Workflow

  1. Submit request via /hidream-i1-full-image-to-image-request endpoint
  2. Receive request_id in response
  3. Poll status endpoint every 2-3 seconds using request_id
  4. When status is COMPLETED, use the image URLs in response
  5. Handle FAILED or ERROR by inspecting error field
