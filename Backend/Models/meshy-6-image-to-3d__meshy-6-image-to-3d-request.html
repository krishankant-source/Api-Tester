topologyNoenumtriangle—Surface topology type. Allowed: quad, triangle. Quad for smooth surfaces, Triangle for detailed geometry.
target_polycountNointeger30000—The desired number of polygons in the generated 3D model. Higher values give a more detailed, heavier mesh; lower values give a simpler, lighter one.
symmetry_modeNoenumauto—Symmetry enforcement mode. Allowed: off, auto, on.
should_remeshNobooleantruetrue, falseRebuilds the 3D model's surface grid for cleaner, more even geometry.
should_textureNobooleantruetrue, falseGenerate texture maps for the 3D model.
enable_pbrNoboolean—true, falseGenerates realistic surface materials (PBR: metallic, roughness, normal maps) on top of the base color, so the model reacts properly to light.
pose_modeNoenum——Enforce a specific pose. Allowed: a-pose, t-pose, or empty for no specific pose.
texture_promptNostring——Text prompt to guide the texturing process.
texture_image_urlNostring——URL of a 2D reference image to guide the texturing process.
enable_riggingNoboolean—true, falseAuto-rig the model as a humanoid character with basic walking and running animations. Best for characters with clearly defined limbs.
rigging_height_metersNofloat1.7—Approximate height of the character in meters. Used only when enable_rigging is true.
enable_animationNoboolean—true, falseApply a predefined animation preset. Requires enable_rigging to be true.
animation_action_idNointeger1001—Animation preset ID from Meshy’s library of 500+ presets. See https://docs.meshy.ai/en/api/animation-library.
enable_safety_checkerNobooleantruetrue, falseTurns on automatic filtering of unsafe or explicit (NSFW) content. Leave enabled unless you have a specific reason to disable it.

Status Workflow

The API is asynchronous. Submit a request and poll the result endpoint using the returned request_id.

Status Values

Status Description
QUEUED Request accepted, waiting to be processed
PROCESSING Being processed by the model
COMPLETED Done — output contains the result
FAILED Failed — check error field
ERROR System error — not charged

Status Flow

Submitted → QUEUED → PROCESSING → COMPLETED/FAILED/ERROR

Typical Workflow

  1. Submit request via POST /meshy-6-image-to-3d-request
  2. Receive response containing request_id
  3. Poll /v2/requests/status/{request_id} every 5-10 seconds
  4. When status is COMPLETED, use the output URLs to download your files

Notes & Tips

  • Processing time is typically 5–10 minutes.
  • For best results, use clear, high-contrast images with well-defined edges and minimal background clutter.
  • The API returns multi-format outputs: GLB, FBX, OBJ, USDZ, BLEND, and STL files.
  • Enable rigging only for humanoid characters with clearly defined limbs.
  • Texture quality depends heavily on input image clarity and lighting conditions.
  • Enable safety checker to prevent generation of inappropriate content (recommended).
