Request Parameters - FIBO generate request

Field Type Required Default Description
prompt string No — Freeform text description for image generation. Use this OR structured_prompt, not both.
structured_prompt object No — JSON scene specification containing fields like objects, lighting, aesthetics, photographic_characteristics, style_medium, etc. Use this OR prompt, not both.
image_url string No — URL of a reference image to inspire generation. Used for style or composition guidance.
negative_prompt string No "" Content to explicitly avoid in the generated image.
aspect_ratio enum No 1:1 Desired output image aspect ratio. Allowed values: 1:1, 2:3, 3:2, 3:4, 4:3, 4:5, 5:4, 9:16, 16:9.
resolution enum No 1MP Output image resolution. Allowed values: 1MP, 4MP.
steps_num integer No 50 Number of inference steps to perform. Higher values may improve detail but increase processing time.
seed integer No 5555 Random seed for reproducible results. Use the same seed with identical prompts to generate identical outputs.
sync_mode boolean No — If true, the API returns the generated image directly in the response instead of returning a request_id. If false or omitted, the response is asynchronous with a request_id.

Status Values

Status Description
QUEUED Request accepted, waiting to be processed
PROCESSING Being processed by the model
COMPLETED Done — output contains the result
FAILED Failed — check the error field for details
ERROR System error occurred — request was not processed and not charged

Status Flow

Submitted → QUEUED → PROCESSING → COMPLETED/FAILED/ERROR
                     ↘ Initial poll every 2-3s

Typical Workflow

  1. Submit request to /fibo-request
  2. Receive request_id and polling_url
  3. Use polling_url to check status with GET request
  4. Poll every 2–3 seconds until status is COMPLETED, FAILED, or ERROR
  5. When COMPLETED, use the output.media_url to retrieve image

Notes & Tips

  1. Use async mode (default) for most applications; use sync_mode: true only for low-latency needs.
  2. Poll the status endpoint every 2–3 seconds until status is COMPLETED, FAILED, or ERROR.
  3. Implement exponential backoff for retry logic on ERROR or 500 responses.
  4. Use structured_prompt for complex scenes requiring precise object placement, lighting, or style.
  5. For reproducible results, fix the seed value and use consistent prompts.
  6. Avoid using both prompt and structured_prompt simultaneously — choose one for clarity.
  7. Image resolution options: 1MP (1024x1024) and 4MP (2048x2048).
  8. Processing time varies based on load and resolution (typically 5–20 seconds).
